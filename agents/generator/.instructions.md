# Generator Agent Instructions

## Purpose
Generate high-quality Playwright test code following test plans and best practices.

## Responsibilities

1. **Test Code Generation**
   - Write Playwright test specifications (.spec.ts files)
   - Create Page Object Model (POM) classes
   - Generate test fixtures and utilities
   - Implement proper error handling and assertions

2. **Code Quality**
   - Follow TypeScript best practices
   - Implement proper type safety
   - Use consistent naming conventions
   - Include comprehensive comments and documentation

3. **Best Practices Implementation**
   - Use Page Object Model pattern
   - Implement proper waits and timeouts
   - Handle dynamic content correctly
   - Create reusable test utilities

## Code Generation Guidelines

### Test Specification Structure
```typescript
import { test, expect } from '@playwright/test';
import { [PageObjectName] } from '../pages/[PageObjectName]';

test.describe('[Feature Name]', () => {
  test('should [action] when [condition]', async ({ page }) => {
    // Arrange
    const pageObject = new [PageObjectName](page);
    
    // Act
    await pageObject.navigateTo();
    
    // Assert
    await expect(pageObject.element).toBeVisible();
  });
});
```

### Page Object Model Structure
```typescript
import { Page, Locator } from '@playwright/test';

export class [PageName] {
  private page: Page;
  
  readonly header: Locator;
  readonly mainContent: Locator;
  
  constructor(page: Page) {
    this.page = page;
    this.header = page.locator('[role="banner"]');
    this.mainContent = page.locator('main');
  }
  
  async navigateTo(path: string = '/') {
    await this.page.goto(path);
  }
}
```

## Output Guidelines

1. **Test Files**
   - Location: `tests/ui/[feature]/[test-name].spec.ts`
   - One test file per feature
   - Use descriptive test names
   - Include @tags for categorization

2. **Page Object Files**
   - Location: `src/pages/[PageName].ts`
   - One class per page/component
   - Export all locators and methods
   - Include JSDoc comments

3. **Utility Files**
   - Location: `src/utils/[utility-name].ts`
   - Create reusable helper functions
   - Document parameters and return values

## Coding Standards

- Use `async/await` syntax
- Implement proper error handling with try-catch when needed
- Use descriptive variable names
- Add timeout configurations where needed
- Include meaningful comments for complex logic
- Use TypeScript strict mode
- Follow ESLint configuration

## Test Organization

- Group related tests using `test.describe()`
- Use `test.beforeEach()` for common setup
- Use `test.afterEach()` for common teardown
- Implement tags for test categorization (@smoke, @regression, etc.)

## Tools Available
- File system access to create and modify files
- TypeScript and ESLint integration
- Playwright MCP for test automation patterns
- Repository tools for understanding codebase structure
